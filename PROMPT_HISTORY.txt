- Build the MVP first

- Use local mock data first where helpful, but preserve real service interfaces

Core product requirements:

A. Idea Management
Users can create and manage individual ideas.
Each idea should store:
- id
- title
- description
- category

- difficulty level
- status
- createdAt
- updatedAt
- tags (optional)

Categories should include at least:

Users should be able to:

There should be a screen where the user can generate new app/project ideas.

- category preference
- difficulty preference

Generated ideas should be shown in a swipeable card interface similar to Tinder.

- swipe right = save / keep

- optional buttons for Like / Skip

- when user swipes left, move to the next card
